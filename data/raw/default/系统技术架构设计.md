# 企业级工单智能分流 RAG 系统 —— 技术架构设计

本文档描述系统的技术实现，用于说明「检索—生成—防幻觉」全链路的工程取舍。

## 1. 整体架构

- 前端：Streamlit（`dashboard/chat.py`），提供对话界面与参考资料展示
- 后端：FastAPI（`main.py` + `app/api`），提供 `/api/rag/query`、`/api/auth/*` 等接口
- 异步：Celery + Redis，负责文档入库、向量化等长任务
- 模型运行时：本地 Ollama（默认），承载 Embedding 与 LLM 两块能力
- 存储：每个租户独立的 FAISS 向量索引 + BM25 倒排 + Redis 会话/缓存

## 2. 混合检索（Hybrid Search）

两路并行召回后再融合，兼顾「语义」与「关键词」：

- **向量检索**：本地 Ollama `bge-m3`（1024 维，多语言/中文 SOTA），文本块 L2 归一化后存入索引，检索用精确内积（等价于余弦相似度）。
- **BM25 关键词检索**：`rank-bm25` 库，中文用 `jieba` 分词，补向量检索在专有名词、编号、公式上的召回短板。
- **加权融合**：`vector_score * 0.6 + bm25_score * 0.4`（经验起步权重，向量主导保证语义召回，BM25 占四成补关键词召回；可用评测集反推调优）。
- **RRF 重排**：对融合后的候选做 Reciprocal Rank Fusion，`score = Σ 1/(k + rank_i)`，`k=60`。RRF 对两套检索的分数量纲差异不敏感，比简单加权更稳健，且**零模型、零延迟、可离线**。相比线上「BGE 交叉编码器重排」，RRF 胜在免费可离线；追求极致精度可在 `.env` 配 `RERANKER=cross_encoder` 接入本地交叉编码器（可选增强）。

## 3. 向量存储（Vector Store）

- 索引结构：`faiss.IndexFlatIP`（内积，配合归一化向量即余弦相似度）。FAISS 用于构建与持久化。
- 检索实现：用 numpy 精确内积矩阵做检索，与 FAISS IndexFlatIP 数学等价；该做法规避了 Windows 平台下 `faiss.search` 原生调用的兼容性问题，同时保证精确召回。
- 多租户：每个租户独立索引目录（`data/vector_db/{tenant}/`），互不干扰。

## 4. 多智能体编排（Agentic RAG）

`AgentOrchestrator` 作为调度中心，先用**规则 + 中文关键词 + 正则**识别意图，再分派给对应 Agent：

- **意图识别（规则）**：优先用规则/关键词路由（可解释、零延迟、零成本），而非 LLM 路由；仅在歧义时回退。
- **RetrievalAgent**：负责检索 + 生成回答，并支持「反思重检索」（首轮答案不确定时，改写 query 再检索一轮）。
- **ToolAgent**：计算类问题（含四则运算、括号）走工具智能体，用确定性计算器得出精确结果，避免 LLM 数值幻觉。
- **HallucinationAgent**：对答案做事实/数字校验，推导 `hallucination_level`（low/medium/high）；高风险时在答案中显式提示「请核对参考资料」，把不确定性交还给用户。

## 5. 会话与缓存

- **会话记忆**：`ConversationMemory` 基于 Redis 存储多轮对话，支持追问承接上下文。
- **租户级缓存**：相同问题（含租户维度）命中 Redis 缓存直接返回，降低重复检索/生成开销。

## 6. 鉴权与多租户隔离

- 鉴权：`python-jose`（JWT HS256）+ SHA256 口令散列。
- 隔离：JWT 携带租户标识 → 路由到该租户独立向量索引；Redis 键按租户前缀隔离。不同租户数据在检索、缓存、历史三层均隔离。

## 7. 完全本地、免费、离线

- Embedding：本地 Ollama `bge-m3`
- LLM：本地 Ollama `qwen2.5:7b`（自动调用 RTX 4050 GPU 加速）
- 重排：本地 RRF（无需模型）
- 整套不依赖任何付费 API、不需要外网、零 token 成本。
- 在线 SiliconFlow API、本地 LoRA 作为**可选**后端保留，按需切换（`LLM_BACKEND`）。

## 8. 评测指标（本地 Ollama 实测）

| 指标 | 结果 | 说明 |
|------|------|------|
| retrieval recall@5 | 1.00 | 检索召回已解决 |
| MRR | 0.935 | 相关结果排位靠前 |
| 答案相关性 | 95% | LLM 裁判打分 |
| 忠实度 | 75.6% | 答案基于检索内容 |
| fact_recall | 0.454 | 关键事实全覆盖率（瓶颈在 7B 生成保真度，已由幻觉检测 + 反思重检索缓解） |

## 9. 部署

- 本地一键：`python start.py`（自动检查并拉取缺失模型、缺失索引自动建库、拉起后端 + 前端 + Celery）
- Docker：`docker-compose up` 编排 redis / rag-api / celery-worker / frontend 四服务，带健康检查
- 前端默认 http://localhost:8501 ，后端 API 文档 http://localhost:8000/docs
